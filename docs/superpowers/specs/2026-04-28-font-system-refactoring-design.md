# デザイン設計書: フォントシステム刷新 (Rust主導への移行)

## 1. 目的
- ブラウザの Local Font Access API への依存を排除し、ユーザーが権限をブロックした場合でもフォント読み込みが不可能になる問題を解決する。
- フォントファイル名ではなく、正確な「フォントファミリー名」を取得し、エディタおよびプレビューで確実にフォントが適用されるようにする。

## 2. アーキテクチャ
- **現状**: フロントエンド (`queryLocalFonts`) とバックエンド (手動ファイルスキャン) のハイブリッド。
- **変更後**: バックエンド (Rust + `fontdb`) による一括管理。フロントエンドはバックエンドから提供されるリストを表示・利用するのみとする。

## 3. 変更詳細

### 3.1 バックエンド (Rust / `src-tauri/src/font_ops.rs`)
- **依存関係の追加**: `fontdb` クレートを導入。
- **スキャンロジックの改善**:
    - `fontdb::Database` を使用してシステムフォントをロード。
    - 各フォントフェイスから `family` 名を取得。
    - ファイル名ベースの取得を廃止し、OSが認識する正式なファミリー名を返すように変更。
- **Tauriコマンド**: `get_os_fonts` および `refresh_os_fonts` を `fontdb` ベースに刷新。

### 3.2 フロントエンド (React / `src/components/Settings.tsx`)
- **API呼び出しの削除**: `window.queryLocalFonts` に関連するロジックをすべて削除。
- **データフローの統一**: フォントリストの更新は `invoke('refresh_os_fonts')` のみに集約。
- **UI/UX**: 
    - フォント読み込み中のインジケーター表示の継続。
    - フォントリストが空の場合のフォールバック (`system-ui`) の保証。

### 3.3 データ仕様 (`変数関数仕様書.md` への反映予定)
- `get_os_fonts`: システムフォントのファミリー名リスト (`Vec<String>`) を返す。
- `refresh_os_fonts`: キャッシュを破棄して再スキャンし、最新のリストを返す。

## 4. アクセシビリティ (WCAG 2.2 Level AA)
- フォント選択リスト (`<select>`) の各項目がスクリーンリーダーで正しく読み上げられることを確認。
- キーボード操作 (矢印キーでの選択、Enterでの決定) がスムーズに行えることを保証。

## 5. テスト・検証計画
- Windows環境でのフォント取得が正常に行われ、日本語フォント名（「ＭＳ ゴシック」等）が正しく取得できるかを確認。
- 取得したフォント名を選択した際、Monaco Editor およびプレビュー画面に正しく反映されるかを目視確認。
