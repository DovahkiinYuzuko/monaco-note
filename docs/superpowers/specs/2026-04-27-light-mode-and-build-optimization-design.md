# デザイン仕様書：ライトモード最適化、ビルド環境構築、およびファイル起動のサポート

**作成日**: 2026-04-27
**ステータス**: ドラフト（承認待ち）

## 1. 概要
モナコノートのライトモードにおける視認性の向上、Windows/macOS両対応のビルド環境の構築、およびOSからのファイル起動（右クリックで開く等）のサポートを行う。

## 2. 目標
- ライトモード時に Markdown プレビューのテーブルやコードブロックが背景色に馴染みつつ、高い可読性を維持すること。
- Windows でインストール EXE (NSIS) を生成し、デスクトップショートカットを自動作成すること。
- macOS でのビルド (DMG/App) をサポートし、クロスプラットフォーム対応を強化すること。
- コマンドライン引数経由で渡されたファイルパスを検知し、アプリ起動時にその内容をエディタで開くこと。

## 3. 詳細仕様

### 3.1. ライトモード・プレビューの改善 (案A: 馴染ませベージュ)
- **GitHub Markdown CSS のオーバーライド**:
    - アプリ全体の背景色 `#faf9f6` に合わせ、テーブルの偶数行背景 (`--color-canvas-subtle`) を `#f2f1ee` などのベージュ系グレーに設定。
    - 文字色は `var(--text-main)` を使用し、コントラストを確保。
- **コードブロック**:
    - ライトモード時、背景色を `#f6f8fa` に変更。
    - `highlight.js` のテーマを、ライトモード時は `github.css` (または相当の配色) に切り替わるよう CSS で制御。
- **UIコンポーネント**:
    - コマンドパレットの影 (`box-shadow`) をライトモード用に淡い色に変更。
    - ステータスバーの背景色を、テーマに応じた `var(--primary)` に統一。

### 3.2. ビルド設定の最適化
- **ターゲット構成**: `tauri.conf.json` の `bundle.targets` を `["nsis", "app", "dmg"]` に設定。
- **NSIS 設定**:
    - `oneClick: false`: ユーザーがインストール先を選択可能にする。
    - `allowToChangeInstallationDirectory: true`: 同上。
    - `createDesktopShortcut: "always"`: デスクトップにショートカットを作成。
- **ライブラリ修正**: `lucide-react` のインポートを `{ IconName } from "lucide-react"` 形式に統一し、TypeScript の型エラーを解消。

### 3.3. ファイル起動サポート
- **Rust 側 (`main.rs`)**:
    - 起動時のコマンドライン引数を解析。
    - ファイルパスが含まれる場合、フロントエンド準備完了後にイベント (例: `open-file`) を emit する。
- **フロントエンド側 (`App.tsx`)**:
    - `open-file` イベントを listen し、渡されたパスを `useTabs` の `openFile` 関数に渡して自動的に開く。

## 4. 依存関係と影響範囲
- **依存関係**: `lucide-react`, `github-markdown-css`, `highlight.js`, `tauri-apps/api`.
- **影響範囲**: アプリケーション全体のテーマ適用、ビルドプロセス、アプリ起動フロー。

## 5. テスト計画
- **表示確認**: ライトモードに切り替え、テーブルおよびコードブロックが正しく表示されるか目視で確認。
- **ビルドテスト**: `npm run tauri build` を実行し、NSIS インストーラーが生成されることを確認。
- **起動テスト**: コマンドラインから `monaco-note.exe <file_path>` を実行し、指定したファイルがエディタで開かれることを確認。
