# デザイン設計書: フォント設定反映の確実化 (クオート対応 & ローカライズ名対応)

## 1. 目的
- フォント名にスペースや特殊文字が含まれる場合に、エディタおよびプレビューで正しく適用されない問題を解決する。
- 日本語名（ローカライズ名）を持つフォントにおいて、ブラウザが期待する名称と Rust 側が取得する名称の不一致を解消する。

## 2. 変更詳細

### 2.1 バックエンド (Rust / `src-tauri/src/font_ops.rs`)
- **多言語名の取得**: 
    - `face.families` の `.first()` だけでなく、全要素を走査してすべての名称（英語名、日本語名等）を取得するように変更。
    - 取得したすべての名称をフラットなリストにし、重複を排除して返す。
- **効果**: ブラウザが「ＭＳ ゴシック」を期待していても「MS Gothic」を期待していても、どちらの名称も選択肢に現れるようにする。

### 2.2 フロントエンド (React / `src/components/Editor.tsx` & `Preview.tsx`)
- **クオート処理の徹底**:
    - Monaco Editor の `options.fontFamily` に渡す際、フォント名をダブルクオート `"` で囲む。
    - Markdown プレビューの `font-family` スタイルに適用する際、同様にダブルクオート `"` で囲む。
- **効果**: スペースを含むフォント名（例: "Cascadia Code"）が CSS プロパティとして正しくパースされるようにする。

## 3. アクセシビリティ (WCAG 2.2 Level AA)
- フォント名の多言語対応により、ユーザーが普段使い慣れている名称でフォントを見つけやすくなる（Understandable の向上）。

## 4. テスト・検証計画
- **検証項目1**: 「ＭＳ ゴシック」や「Meiryo」などの日本語フォントを選択し、エディタとプレビューの両方で正しく切り替わるか。
- **検証項目2**: 「Courier New」や「Segoe UI」などのスペースを含むフォントを選択し、正しく適用されるか。
- **検証項目3**: 開発者ツールで各要素に適用されている `font-family` がクオートで囲まれているか確認。
